    对于原型的基本知识之上，更加深入一些。

    原型基础

    首先得明确，prototype和__proto__的意思和他们扮演的角色

    prototype是构造函数指向属于自己的原型的（此时的视角为构造函数）

    __proto__是实例化对象指向构造自己的函数的原型的（此时的视角为实例化对象）

    在明确了以上概念，下面讨论js的native code

    Function 和 Object

    在我们使用自己的构造函数的时候，比如 function Person( ){..} ，实际上他执行的是 let Person = new Fucntion..，这意味着Person这个对象是Function的实例化对象。那么问题来了，Function作为一个构造函数他肯定也是通过类似let Function = …来实现的，所以Function的__proto__指向的是谁呢，难道还有别的内置函数声明了Function？

    事实上Function.prototype.__proto__===Function，意味着Function的构造函数是他自己！也就是自己new自己出来的，所以Function的隐式原型和显式原型都是Function.prototype这个对象。

    那么来看另一个内置对象Object，Object实际上是通过new Function()构造出来的，所以Object的隐式原型指向Function.prototype，而Object.prototype的__proto__为null，也就是原型链的末尾。
